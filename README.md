# UAV_project

P600 UAV flight-control-side workspace. This repository contains the PX4/MAVROS execution chain, localization integration, mocap bridge, and UAV state management. It consumes the unified experiment references generated by [p600_arm_control](https://github.com/chnchenfan/p600_arm_control).

GitHub: <https://github.com/chnchenfan/P600_uav_control>  
Companion arm repository: <https://github.com/chnchenfan/p600_arm_control>

## Overview

- Recommended environment: Ubuntu 18.04 + ROS Melodic
- The main flight package is `src/uav`
- Mocap integration is under `src/vrpn_client_ros`
- T265/ToF related localization components are under `src/t265_tof`
- Launch and helper scripts live in `shell/`

Key directories:

- `src/uav/launch`: UAV control and localization launch files
- `src/uav/src`: flight execution, guideflight, and state handling code
- `src/vrpn_client_ros/launch`: VRPN client and mocap-to-MAVROS bridge launch files
- `shell/Simulation`: simulation startup scripts
- `shell/Experiment`: experiment helper scripts

## Information Flow

This repository is the flight execution layer. It does not generate experiment trajectories by itself. Instead, it consumes `/wjl/guidefly/pose_d` produced by the task layer in `p600_arm_control`.

1. Experiment reference input

`p600_arm_control` publishes:

- `/wjl/guidefly/pose_d`

The message carries:

- `x_d`
- `y_d`
- `z_d`
- `yaw_d`
- `land_flag`

2. UAV execution chain

- `src/uav/launch/desired_uav_fly.launch` starts:
  - `Driver0`
  - `desired_uav_fly`
- `src/uav/src/uav/Uav_info.cpp` subscribes to `/wjl/guidefly/pose_d`
- Incoming position, yaw, and landing commands are converted into the UAV execution path through PX4/MAVROS

3. UAV feedback

The flight side continuously provides:

- `/mavros/local_position/pose`
- `/mavros/state`

These feedback signals are also used by the experiment nodes in `p600_arm_control`.

4. Mocap and external localization

The mocap entrypoint is:

- `src/vrpn_client_ros/launch/vrpn_wjl.launch`

The default chain is:

- `vrpn_client_node` publishes `/vrpn_client_node/Tracker0/pose`
- `vrpn2mavros` bridges mocap to `/mavros/vision_pose/pose`

For experiment 4, the system also needs:

- `/vrpn_client_node/ring1/pose`
- `/vrpn_client_node/ring2/pose`
- `/vrpn_client_node/ring3/pose`
- `/vrpn_client_node/ring4/pose`

These ring topics are provided by the mocap system and consumed by `p600_arm_control`.

## The Four Experiments

The task logic for all four experiments lives in `p600_arm_control`. In this repository, the role stays consistent: receive `/wjl/guidefly/pose_d` and execute the UAV motion.

### Experiment 1: Hover with arm disturbance baseline

This repository is responsible for:
- takeoff, OFFBOARD entry, and hover tracking
- following the fixed base reference published by the arm-side experiment node

Flight entrypoint:
- `src/uav/launch/desired_uav_fly.launch`

### Experiment 2: Circular base motion with end-effector hold compensation

This repository is responsible for:
- tracking the circular base reference from the arm-side experiment node
- maintaining position and yaw stability while the arm side performs IK compensation

Flight entrypoint:
- `src/uav/launch/desired_uav_fly.launch`

### Experiment 3: Square base motion with periodic arm motion

This repository is responsible for:
- tracking square base references
- providing `mavros/state` and `local_position/pose` feedback for safety monitoring on the arm side

Flight entrypoint:
- `src/uav/launch/desired_uav_fly.launch`

### Experiment 4: Aerial ring passage

This repository is responsible for:
- tracking the base reference generated for the ring-passage task
- continuing to provide flight state and pose feedback
- supplying the mocap environment for the UAV rigid body and rings

Flight entrypoints:
- `src/uav/launch/desired_uav_fly.launch`
- Mocap entrypoint: `src/vrpn_client_ros/launch/vrpn_wjl.launch`

## Common Entrypoints

Flight execution:

- `src/uav/launch/desired_uav_fly.launch`

Localization and parameter loading:

- `src/uav/launch/uav_yaml.launch`

Mocap bridge:

- `src/vrpn_client_ros/launch/vrpn_wjl.launch`

Dynamic manual guideflight:

- `src/uav/launch/dynamic_guidefly.launch`

## Division of Responsibility with p600_arm_control

`UAV_project` is responsible for:

- UAV state acquisition
- PX4/MAVROS flight execution
- localization integration
- mocap-to-MAVROS bridging

`p600_arm_control` is responsible for:

- arm execution
- experiment 1/2/3/4 task logic
- generation of `/wjl/guidefly/pose_d` and `/wjl/arm/guidefly/angle_d`
- recording and experiment plotting

The main system chain is:

`experiment node -> /wjl/guidefly/pose_d -> UAV_project flight execution -> /mavros/* feedback -> experiment node`

## Notes

- This repository does not split into dedicated flight nodes per experiment; experiments 1/2/3/4 reuse the same UAV execution path.
- If ring topics are not automatically published by `vrpn_client_node`, tracker names such as `ring1` to `ring4` need to be exposed through the mocap configuration.

# UAV_project

P600 无人机飞行控制侧工作空间。这个仓库负责 PX4/MAVROS 侧飞行执行、定位接入、动捕桥接和无人机状态管理，并接收 [p600_arm_control](https://github.com/chnchenfan/p600_arm_control) 发布的统一实验期望。

GitHub: <https://github.com/chnchenfan/P600_uav_control>  
配套机械臂仓库: <https://github.com/chnchenfan/p600_arm_control>

## 基本信息

- 推荐运行环境: Ubuntu 18.04 + ROS Melodic
- 飞行控制主包位于 `src/uav`
- 动捕接入位于 `src/vrpn_client_ros`
- T265/ToF 相关定位组件位于 `src/t265_tof`
- 启动与辅助脚本位于 `shell/`

核心目录:

- `src/uav/launch`: 无人机控制与定位相关 launch
- `src/uav/src`: 飞行控制、模式切换、期望跟踪相关代码
- `src/vrpn_client_ros/launch`: 动捕客户端与动捕到 MAVROS 的桥接入口
- `shell/Simulation`: 仿真启动脚本
- `shell/Experiment`: 实验辅助脚本

## 信息流

这个仓库的角色是“飞行执行层”。它不生成实验轨迹，而是消费任务层给出的 `/wjl/guidefly/pose_d`。

1. 任务期望输入

来自 `p600_arm_control` 的实验节点统一发布：

- `/wjl/guidefly/pose_d`

消息内容包括：

- `x_d`
- `y_d`
- `z_d`
- `yaw_d`
- `land_flag`

2. 无人机执行链

- `src/uav/launch/desired_uav_fly.launch` 启动：
  - `Driver0`
  - `desired_uav_fly`
- `src/uav/src/uav/Uav_info.cpp` 订阅 `/wjl/guidefly/pose_d`
- 收到新目标后，将位置与偏航目标转成飞行指令，并根据 `land_flag` 进入降落收尾流程

3. 无人机反馈

飞控执行依赖并持续读取：

- `/mavros/local_position/pose`
- `/mavros/state`

这些反馈同时也被机械臂侧实验节点用于闭环控制和保护判断。

4. 动捕与外部定位输入

动捕入口位于:

- `src/vrpn_client_ros/launch/vrpn_wjl.launch`

当前默认链路:

- `vrpn_client_node` 输出 `/vrpn_client_node/Tracker0/pose`
- `vrpn2mavros` 将动捕位置桥接到 `/mavros/vision_pose/pose`

在实验四中，还需要额外提供:

- `/vrpn_client_node/ring1/pose`
- `/vrpn_client_node/ring2/pose`
- `/vrpn_client_node/ring3/pose`
- `/vrpn_client_node/ring4/pose`

这些挂环话题由动捕系统产生，实验四控制节点在 `p600_arm_control` 中消费。

## 四个实验

这四个实验的“任务逻辑”都在 `p600_arm_control`，本仓库在四个实验中的职责保持一致：接收 `/wjl/guidefly/pose_d`，执行飞行。

### 实验 1: 悬停下机械臂扰动基线

本仓库角色:
- 起飞、进入 OFFBOARD、悬停跟踪固定目标点
- 跟随机械臂侧发布的固定基座参考

配套飞行入口:
- `src/uav/launch/desired_uav_fly.launch`

### 实验 2: 基座画圆 + 末端固定点补偿

本仓库角色:
- 跟踪机械臂侧发布的圆轨迹基座参考
- 保持位置和偏航稳定，供机械臂侧做末端固定点补偿

配套飞行入口:
- `src/uav/launch/desired_uav_fly.launch`

### 实验 3: 基座方形轨迹 + 机械臂周期运动

本仓库角色:
- 跟踪正方形轨迹基座参考
- 提供 `mavros/state` 和 `local_position/pose` 反馈，供机械臂侧保护逻辑使用

配套飞行入口:
- `src/uav/launch/desired_uav_fly.launch`

### 实验 4: 空中穿环

本仓库角色:
- 跟踪机械臂侧为穿环任务生成的基座参考
- 继续向系统提供飞行状态与位姿反馈
- 提供动捕基座真值与挂环动捕接入环境

配套飞行入口:
- `src/uav/launch/desired_uav_fly.launch`
- 动捕入口: `src/vrpn_client_ros/launch/vrpn_wjl.launch`

## 常用入口

飞行控制:

- `src/uav/launch/desired_uav_fly.launch`

- `src/uav/launch/uav_yaml.launch`

动捕桥接:

- `src/vrpn_client_ros/launch/vrpn_wjl.launch`

动态调参/手工指点:

- `src/uav/launch/dynamic_guidefly.launch`

## 与 p600_arm_control 的分工

`UAV_project` 负责:

- 飞行器状态获取
- PX4/MAVROS 执行链
- 定位接入
- 动捕到 MAVROS 的桥接

`p600_arm_control` 负责:

- 机械臂控制
- 四个实验的任务逻辑
- `/wjl/guidefly/pose_d` 与 `/wjl/arm/guidefly/angle_d` 的生成
- 录包与实验绘图

两仓配合后的主链路是：

`实验节点 -> /wjl/guidefly/pose_d -> UAV_project 飞行执行 -> /mavros/* 反馈 -> 实验节点`

## 说明

- 本仓库本身不区分实验 1/2/3/4 的专用控制节点，四个实验默认共用同一条无人机执行链。
- 如果实验四的挂环话题没有由 `vrpn_client_node` 自动发布，需要在 `vrpn_client_ros` 配置中显式加入 `ring1` 到 `ring4` 的 tracker。
